# Contributing

Thank you for your interest in improving this skill. Here is how to contribute.

## How to Submit Changes

1. Fork the repository.
2. Create a feature branch: `git checkout -b feature/your-improvement`
3. Make your changes to `marketing-research/SKILL.md`.
4. Test the skill by installing it in Claude.ai or Claude Code and running a full research.
5. Submit a pull request with a clear description of what changed and why.

## Writing Standards

All contributions to SKILL.md must follow these rules:

- **Zero em dashes.** Use periods, commas, colons, or separate sentences.
- **Minimal parenthetical content.** If something matters, give it a full sentence.
- **Active voice.** Short paragraphs. Plain language. No hedging.
- **Tables over prose** for all structured comparisons and lists.
- **Point-first writing.** Lead every section with the sharpest insight.

## What Makes a Good Contribution

**Improvements to module instructions** that produce better, more specific, or more actionable output from Claude.

**New mental models** added to Module 5 that unlock genuinely different copy angles.

**Refinements to output format rules** that make the research output more useful for copywriters and marketers.

**Bug fixes** where the skill instructions produce unexpected or incorrect formatting.

## What to Avoid

- Adding em dashes or parenthetical asides to the skill file.
- Making the skill longer without making it better. Brevity is a feature.
- Adding modules that duplicate what existing modules already cover.
- Changing the output format in ways that break the table-first structure.

## Testing Your Changes

Before submitting a pull request:

1. Install the modified skill in Claude.ai or Claude Code.
2. Run a full research for any product with 3-4 competitors.
3. Verify that the output follows all writing rules: zero em dashes, zero parenthetical asides, tables for structured content, point-first writing.
4. Confirm the full output is under 700 lines.

## Questions

Open an issue if you have questions about contributing. Include the specific module or section you want to improve.
